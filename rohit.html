1. Problems in Nature

  • Iteration: Problems with repeated actions (e.g., calculating factorial).
  • Recursion: Problems broken into smaller subproblems (e.g., Fibonacci).
  • Backtracking: Explore solutions to find the best one (e.g., N-Queens).

2. Space and Time Efficiency

  • Time Efficiency: Measures algorithm speed.
  • Space Efficiency: Measures memory usage.

3. Design Principles

  • Divide and Conquer: Break problems into subproblems and combine results.
  • Dynamic Programming: Reuse solutions of overlapping subproblems.
  • Greedy Approach: Make locally optimal choices.
  • Brute Force: Explore all possible solutions.

4. Hierarchical Data and Tree Structures

  • Binary Search Tree: Efficient searching and insertion.
  • Heap: Used for priority queues.
  • Trie: Specialized for prefix searches.

5. Need for Array Query Algorithms

Array query algorithms optimize frequent operations (e.g., finding range sums).

  • Segment Trees: Efficient for range queries.
  • Fenwick Trees: For cumulative frequency computations.

6. Tree vs. Graph Traversals

Tree
Hierarchical structure (e.g., file systems). Traversals: In-order, Pre-order, Post-order.
Graph
General structure with nodes and edges (e.g., social networks). Traversals: BFS, DFS.

7. Sorting and Searching Algorithms

  • Sorting: Merge Sort, Quick Sort, Heap Sort.
  • Searching: Binary Search, Hashing.
